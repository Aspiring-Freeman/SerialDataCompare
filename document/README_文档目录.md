# 文档目录说明

本目录包含串口数据分析工具的所有文档。

## 文档分类

### 1. 用户文档
- **功能说明.md** - 软件功能详细说明
- **README_YYYYMMDD.md** - 项目说明文档（带日期版本）
- **QUICKSTART_YYYYMMDD.md** - 快速开始指南（带日期版本）

### 2. 开发文档
- **REFACTOR_SUMMARY_YYYYMMDD.md** - 重构总结（带日期版本）
- **TODO_YYYYMMDD.md** - 开发任务清单（带日期版本）
- **架构设计_YYYYMMDD.md** - 系统架构设计文档（如有）

### 3. 变更日志
- **CHANGELOG_YYYYMMDD.md** - 版本变更记录（带日期版本）

## 命名规范

### 静态文档
不随时间变化的文档，直接使用描述性名称：
- `功能说明.md`
- `API参考.md`

### 版本文档
随时间变化或版本迭代的文档，使用以下格式：
- `文档名_YYYYMMDD.md` - 例如：`README_20251101.md`
- `文档名_vX.Y.md` - 例如：`API参考_v1.0.md`

### 日志文档
记录日常开发或测试的文档：
- `开发日志_YYYYMMDD.md`
- `测试记录_YYYYMMDD.md`
- `问题记录_YYYYMMDD.md`

## 文档更新流程

1. **创建新文档**
   - 确定文档类型（用户/开发/日志）
   - 按命名规范创建文件
   - 在本目录说明中更新文档列表

2. **更新现有文档**
   - 静态文档：直接修改
   - 版本文档：创建新版本（带新日期），保留旧版本作为历史记录

3. **归档文档**
   - 创建 `archive/` 子目录
   - 将过期文档移入归档目录
   - 保持文档的可追溯性

## 注意事项

- 所有文档使用UTF-8编码
- 使用Markdown格式编写
- 重要变更应同时更新相关文档
- 定期清理过期文档到归档目录
